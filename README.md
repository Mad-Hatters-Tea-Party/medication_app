Explanation of Tables in the database 

User Table (user)
This table describes the user 
Columns:
user_id: Primary key for each user ( this is a string up to 25 characters that the user creates upon registration ) 
user_email: User's email address (optional) 
user_phone: User's phone number (optional). 
user_pwd: User's password (Required). (this is a string between 8 and 45 characters)
user_gender: User's gender, where 0 = female, 1 = male (nullable).
user_dob: Date of birth of the user (Required) The user must be 18 years old or more
user_height: User's height in inches (Required upon creation of table - optional update) Must be between 1 and 95 inches
user_weight: User's weight in pounds (Required upon creation of table - optional update). Must be between 1 and 700 pounds 
created_at: Timestamp of when the user was created.
updated_at: Timestamp of the last update made to the user data.

Relationships:
notifications: One-to-many relationship with the Notification table.
prescriptions: One-to-many relationship with the Prescription table.
side_effects: One-to-many relationship with the SideEffect table.

Medication Table (medication)
The Medication table stores the medications that the user can select from. 
The user cannot change this data. The data is changed only from the database side. 
Columns:
medication_id: Primary key for each medication. 
medication_name: Name of the medication (nullable).
medication_use: Description of the medication's use (nullable). 
Relationships:
side_effects: One-to-many relationship with the SideEffect table.
prescription_details: One-to-many relationship with the PrescriptionDetail table.

Notification Table (notification)
The Notification table contains notifications sent to users, such as reminders or prescription refill alerts.
Columns:
notification_id: Primary key for each notification. (this id number is auto generated by the database system)
user_id: Foreign key referencing User table. 
notification_type: Type of notification (e.g., 1 = refill, 2 = reminder).
notification_message: Text message for the notification (nullable). string value
notification_date: Date and time for the notification to be sent (nullable).
notification_status: Status of the notification (0 = sent, 1 = read). The status is null upon creation. The status is changed through updates
Timestamps:
created_at: Timestamp for when the notification was created.
updated_at: Timestamp for when the notification was last updated.
Relationships:
user: Many-to-one relationship with the User table

Prescription Table (prescription)
The Prescription table stores prescription records associated with users. A prescription can have a start date, end date, and status (active or archived).
Columns:
prescription_id: Primary key, unique identifier for each prescription. (this id number is auto generated by the database system)
prescription_date_start: Start date of the prescription (nullable). 
prescription_date_end: End date of the prescription (nullable).
prescription_status: Status of the prescription (0 = active, 1 = archived).
user_id: Foreign key referencing User table. Indicates the user to whom the prescription belongs to.
Relationships:
user: Many-to-one relationship with the User table.
prescription_details: One-to-many relationship with the PrescriptionDetail table

PrescriptionDetail Table (prescription_detail)
The PrescriptionDetail table contains details for each medication prescribed as part of a prescription. Each record links a specific prescription to a medication. prescription_id and medication_id is the primary key pair
Columns:
prescription_id: Key referencing Prescription table. Indicates the prescription to which the medication belongs.
medication_id: Key referencing Medication table. Indicates which medication is prescribed.
presc_dose: Dosage for the medication (optional). string value 
presc_qty: Quantity of the medication prescribed (optional).  integer value
presc_type: Type of dosage unit (e.g., Grams, Milligrams, Drops). string value
presc_frequency: Frequency of dosage (e.g., once a day, every 6 hours). string value
Relationships:
medication: Many-to-one relationship with the Medication table.
prescription: Many-to-one relationship with the Prescription table.

SideEffect Table (side_effect)
The SideEffect table stores records of side effects experienced by users from medications.
Columns:
side_effects_id: Primary key for each side effect record. (this id number is auto generated by the database system)
user_id: Foreign key referencing User table. Indicates which user reported the side effect.
medication_id: Foreign key referencing Medication table. Indicates which medication caused the side effect.
side_effect_desc: Description of the side effect (optional). string upto 255 characters 
Timestamps:
created_at: Timestamp for when the side effect was recorded.
updated_at: Timestamp for when the side effect record was last updated.
Relationships:
user: Many-to-one relationship with the User table.
medication: Many-to-one relationship with the Medication table
